## 1. Введение и назначение

### 1.1. Цель документа

Настоящее руководство предназначено для преподавателей профессионального образовательного учреждения, работающих по специальности 15.02.14 «Оснащение средствами автоматизации технологических процессов и производств» и использующих интеллектуальную систему-тьютора (далее — **ИИ-тьютор**) в учебном процессе. Документ описывает порядок работы с системой, методические аспекты её применения и рекомендации по эффективному использованию.

### 1.2. Что такое ИИ-тьютор?

ИИ-тьютор — это программный комплекс, интегрированный в среду обучения Moodle, который на основе лекционных и методических материалов преподавателя автоматически создаёт учебно-контролирующие материалы:

- **Краткие конспекты** лекций с выделением ключевых понятий;
- **Тестовые задания** (вопросы с вариантами ответов, открытые вопросы, задачи);
- **Объяснения** сложных тем простым и доступным языком;
- **Дополнительные материалы** для самостоятельного изучения.

### 1.3. Преимущества для преподавателя

| Преимущество | Описание |
|---|---|
| **Экономия времени** | Сокращение времени на подготовку контрольных материалов на 40–60 % |
| **Единство требований** | Единообразие стиля и структуры создаваемых материалов |
| **Дифференциация** | Быстрая генерация вариантов заданий разного уровня сложности |
| **Актуализация** | Возможность оперативного обновления материалов при изменении программы |
| **Аналитика** | Получение данных об освоении студентами отдельных тем и разделов |
| **Индивидуализация** | Создание адаптивных материалов для студентов с разным уровнем подготовки |
| **Сохранение экспертизы** | Преподаватель остаётся экспертом: ИИ лишь помогает автоматизировать рутину |

### 1.4. Основной принцип системы

> **Преподаватель — автор содержания. ИИ — инструмент автоматизации.**

Система не заменяет педагога, а берёт на себя техническую и формализованную часть работы: структурирование, генерацию тестовых заданий и повторение материала. Принципиальные дидактические решения, отбор содержания и контроль качества остаются за преподавателем.

## 2. Обзор системы

### 2.1. Архитектура системы

ИИ-тьютор работает на базе локального сервера колледжа (on-premise), что обеспечивает безопасность данных и независимость от внешних интернет-сервисов. Все учебные материалы и персональные данные остаются внутри информационной системы образовательного учреждения.

**Схема взаимодействия компонентов системы:**

```
Преподаватель (Moodle)
        │
        ▼
  Загрузка лекционного
      материала (.txt)
        │
        ▼
  Moodle LMS (сервер колледжа)
        │
        ▼
  REST API — обработка запроса
        │
        ▼
  ИИ-модель (локальный сервер)
  ┌─────────────────────────┐
  │  Анализ текста          │
  │  Извлечение терминов    │
  │  Генерация конспекта    │
  │  Создание тестов        │
  │  Формирование пояснений │
  └─────────────────────────┘
        │
        ▼
  Ответ через API → Moodle
        │
        ▼
  Результат:
  • Конспект лекции
  • Тест (5–15 вопросов)
  • Пояснения к сложным темам
        │
        ▼
  Преподаватель: проверка,
  редактирование, публикация
        │
        ▼
  Студент: изучение,
  прохождение теста
```

### 2.2. Компоненты системы

| Компонент | Назначение |
|---|---|
| **Moodle LMS** | Основная образовательная платформа; обеспечивает хранение материалов, управление курсами, учёт достижений |
| **Модуль ИИ-тьютора** | Плагин Moodle, обеспечивающий взаимодействие с ИИ-моделью через API |
| **REST API** | Интерфейс обмена данными между Moodle и ИИ-моделью |
| **ИИ-модель** | Локально развёрнутая языковая модель, выполняющая анализ текста и генерацию учебных материалов |
| **База данных** | Хранение сгенерированных материалов, результатов тестирования и аналитических данных |

### 2.3. Требования к рабочему месту преподавателя

- Рабочий компьютер с доступом к внутренней сети колледжа;
- Браузер: Google Chrome (версия 90+), Mozilla Firefox (версии 88+) или Яндекс.Браузер;
- Аккаунт преподавателя в Moodle с правами редактирования курса;
- Доступ в интернет **не требуется** для работы ИИ-тьютора (система работает на локальном сервере).

## 3. Подготовка материалов для обработки ИИ

Качество сгенерированных материалов напрямую зависит от качества исходного текста. Чем лучше структурирован лекционный материал, тем точнее и полезнее будет результат.

### 3.1. Рекомендуемые форматы материалов

| Формат | Рекомендация |
|---|---|
| **Формат .txt** (основной) | Предпочтительный формат. Простой текст без форматирования. |
| **PDF → конвертация в .txt** | Допустимо. Перед загрузкой проверьте результат конвертации. |
| **Microsoft Word → сохранение как .txt** | Допустимо. Используйте кодировку UTF-8. |
| **Markdown (.md)** | Допустимо. Заголовки и списки помогут ИИ лучше структурировать результат. |

### 3.2. Правила оформления лекционного материала

**Рекомендуется:**

1. **Чёткая структура** — разделяйте материал на смысловые блоки (темы, разделы, подразделы);
2. **Заголовки** — помечайте заголовки строками без абзацного отступа или символом `#`;
3. **Определения** — выделяйте определения ключевых терминов. Пример: «*Датчик — устройство, преобразующее измеряемую величину в сигнал*»;
4. **Нумерованные и маркированные списки** — ИИ корректно обрабатывает перечисления (классификации, этапы процессов, характеристики оборудования);
5. **Примеры и задачи** — включайте конкретные примеры из практики промышленной автоматики (типы датчиков, характеристики контроллеров, схемы автоматизации);
6. **Объём** — оптимальный размер одного материала: от 2 000 до 10 000 символов (примерно 1–3 страницы текста). Слишком короткий текст не даст ИИ достаточно контекста; слишком длинный — может привести к потере деталей.

**Не рекомендуется:**

1. Вставлять таблицы в формате Word — конвертация в .txt разрушит их структуру. Перепишите таблицу в виде текстового списка;
2. Использовать большое количество изображений — ИИ работает с текстом, а не с графикой;
3. Загружать сканы документов (фотографии, отсканированные страницы) — система не распознаёт изображения;
4. Включать материалы, содержащие личные данные студентов или иные конфиденциальные сведения;
5. Использовать обилие сокращений без расшифровки — ИИ может неверно интерпретировать аббревиатуры (ПИД-регулятор следует указать как «пропорционально-интегрально-дифференциальный (ПИД) регулятор»).

### 3.3. Пример хорошо подготовленного материала

```
Тема: Измерительные преобразователи

1. Определение
Измерительный преобразователь — техническое средство, служащее для
преобразования измеряемой величины в форму, удобную для обработки,
хранения или передачи.

2. Классификация по виду выходного сигнала
2.1. Электрические (токовый, напряжений, частотный)
2.2. Пневматические
2.3. Гидравлические

3. Токовый преобразователь 4–20 мА
Токовый преобразователь с унифицированным выходным сигналом 4–20 мА
является стандартом в промышленной автоматизации. Ток 4 мА
соответствует нижней границе диапазона измерений (0%), ток 20 мА —
верхней границе (100%). Преимущества: помехоустойчивость, дальность
передачи сигнала до 2000 м.

4. Пример расчёта
Для термопары типа К с диапазоном 0–1000 °C выходной ток 12 мА
соответствует температуре: T = (12 - 4) / (20 - 4) × 1000 = 500 °C.
```

### 3.4. Контрольный список перед загрузкой

- [ ] Текст сохранён в кодировке UTF-8
- [ ] Объём текста — от 2 000 до 10 000 символов
- [ ] В тексте есть заголовки и разделение на блоки
- [ ] Термины и аббревиатуры расшифрованы
- [ ] Таблицы переписаны в текстовом формате
- [ ] Отсутствуют изображения и сканы
- [ ] Отсутствуют персональные данные

## 4. Работа с системой в Moodle

### 4.1. Доступ к модулю ИИ-тьютора

1. Войдите в Moodle под своей учётной записью преподавателя;
2. Перейдите на страницу нужного курса (например, «Промышленная автоматизация — группа ТЗ-221»);
3. В блоке «Настройки» выберите **«ИИ-тьютор»** или откройте соответствующий раздел курса;
4. Выберите нужный инструмент: **«Генерация конспекта»**, **«Генерация теста»** или **«Пояснение темы»**.

### 4.2. Загрузка лекционного материала и генерация конспекта

1. **Нажмите кнопку** «Загрузить материал»;
2. **Выберите файл** в формате .txt с вашего компьютера;
3. **Укажите параметры генерации:**
   - Дисциплина: «Промышленная автоматизация» или «Измерительные системы»;
   - Уровень сложности: базовый / средний / продвинутый;
   - Требуемый объём конспекта: краткий (до 500 слов) / развёрнутый (до 1500 слов);
4. **Нажмите** «Сгенерировать»;
5. **Дождитесь результата** — обычно генерация занимает от 15 до 60 секунд. На экране появится индикатор прогресса;
6. **Просмотрите** сгенерированный конспект в окне предпросмотра.

### 4.3. Генерация тестовых заданий

1. **Загрузите лекционный материал** (или выберите ранее загруженный);
2. **Выберите тип теста:**
   - Тест с вариантами ответов (один правильный ответ);
   - Тест с множественным выбором (несколько правильных ответов);
   - Открытые вопросы (ввод ответа текстом);
   - Комбинированный тест (смесь типов вопросов);
3. **Настройте параметры:**
   - Количество вопросов: от 5 до 20;
   - Уровень сложности: базовый / средний / продвинутый / смешанный;
   - Включить пояснения к ответам: да / нет (рекомендуется «да» — студент увидит объяснение после ответа);
4. **Нажмите** «Сгенерировать тест»;
5. **Проверьте** сгенерированные вопросы (подробности — в разделе 5).

### 4.4. Генерация пояснений к сложным темам

1. **Укажите тему** для пояснения в текстовом поле (например, «Принцип работы ПИД-регулятора»);
2. **Опишите целевую аудиторию:** студенты 2 курса / студенты 3 курса;
3. **При необходимости загрузите** дополнительный материал для контекста;
4. **Нажмите** «Сгенерировать пояснение»;
5. **Результат** — структурированное объяснение темы с примерами из промышленной автоматики.

### 4.5. Публикация материалов для студентов

После проверки и редактирования:

1. **Нажмите** «Опубликовать» — материал станет доступен студентам в соответствующем разделе курса;
2. **Или** «Сохранить как черновик» — материал будет доступен только преподавателю до завершения редактирования;
3. **Укажите сроки доступности** (необязательно): дата открытия и дата закрытия для студентов.

## 5. Проверка и редактирование результатов ИИ

ИИ-тьютор — это **помощник, а не замена преподавателя**. Все сгенерированные материалы подлежат обязательной проверке перед публикацией.

### 5.1. Чек-лист проверки конспекта

| № | Что проверить | Почему это важно |
|---|---|---|
| 1 | **Точность фактов** — соответствуют ли утверждения учебной литературе | ИИ может непреднамеренно исказить данные (например, диапазон измерений датчика) |
| 2 | **Терминология** — используется ли корректная отраслевая терминология | Проверьте соответствие ГОСТ, учебным программам и рабочим тетрадям |
| 3 | **Полнота** — все ли ключевые темы исходного материала отражены | ИИ может пропустить менее очевидные, но важные аспекты |
| 4 | **Последовательность** — логичен ли порядок изложения | ИИ может переставить разделы, нарушая методическую логику |
| 5 | **Уровень сложности** — соответствует ли материал уровню подготовки студентов СПО | ИИ может излишне упростить или, наоборот, усложнить изложение |
| 6 | **Примеры** — корректны ли числовые примеры и расчёты | Проверьте все формулы, единицы измерения и арифметику |
| 7 | **Отсутствие ошибочных утверждений** (галлюцинаций) | ИИ может сгенерировать правдоподобные, но неверные факты |

### 5.2. Чек-лист проверки тестовых заданий

| № | Что проверить | Пример возможной ошибки |
|---|---|---|
| 1 | **Корректность правильного ответа** | Все варианты ответов неверны, или несколько верных при указании одного |
| 2 | **Однозначность формулировки** | Вопрос допускает двойное толкование |
| 3 | **Отсутствие подсказок** в неправильных вариантах ответа | Длина правильного ответа значительно больше остальных |
| 4 | **Соответствие материалу** | Вопрос относится к теме, не освещённой в лекции |
| 5 | **Корректность пояснений** | Объяснение к ответу содержит фактическую ошибку |
| 6 | **Баланс сложности** | Все вопросы одного уровня; отсутствуют как простые, так и сложные |
| 7 | **Орфография и пунктуация** | Грамматические ошибки в формулировках вопросов |

### 5.3. Инструменты редактирования

Встроенный редактор Moodle позволяет:

- **Редактировать текст** — исправлять ошибки, уточнять формулировки, добавлять недостающую информацию;
- **Удалять вопросы** — убирать некорректные или избыточные тестовые задания;
- **Добавлять вопросы вручную** — создавать собственные вопросы в дополнение к сгенерированным;
- **Изменять порядок** — переставлять вопросы и разделы конспекта;
- **Форматировать текст** — использовать жирный шрифт, списки, заголовки для лучшей читаемости;
- **Вставлять ссылки** — добавлять ссылки на внешние ресурсы или другие разделы курса;
- **Сохранять версии** — каждое сохранение создаёт новую версию, к которой можно вернуться.

### 5.4. Процесс утверждения

Рекомендуемый порядок работы:

1. **Генерация** — ИИ создаёт черновик;
2. **Первая проверка** — преподаватель читает материал и отмечает проблемные места;
3. **Редактирование** — внесение исправлений и дополнений;
4. **Вторая проверка** (при необходимости) — повторное чтение отредактированного материала;
5. **Публикация** — материал становится доступен студентам.

## 6. Мониторинг активности студентов

### 6.1. Доступная аналитика

ИИ-тьютор собирает и отображает следующие данные:

| Показатель | Описание |
|---|---|
| **Количество просмотров конспекта** | Сколько студентов открыли материал |
| **Время изучения** | Среднее время, которое студент тратит на изучение конспекта |
| **Результаты тестирования** | Баллы по каждому студенту и каждому тесту |
| **Количество попыток** | Сколько раз студент проходил тест |
| **Темы с низким результатом** | Вопросы и темы, по которым студенты допускают наибольшее количество ошибок |
| **Активность во времени** | Распределение обращений к материалам по дням и часам |
| **Индивидуальная аналитика** | Детальная статистика по каждому студенту |

### 6.2. Как просмотреть аналитику

1. Откройте курс в Moodle;
2. Выберите **«ИИ-тьютор» → «Аналитика»**;
3. Выберите интересующий период (неделя, месяц, семестр, произвольный);
4. Выберите тип отчёта: по группе, по студенту, по теме, общий;
5. Нажмите **«Сформировать отчёт»**;
6. Отчёт можно просмотреть на экране или экспортировать в формате CSV для дальнейшей обработки.

### 6.3. Как использовать аналитику в работе

- **Выявление сложных тем** — если по определённому тесту средний балл группы ниже 60 %, материал требует доработки или дополнительного разбора на занятии;
- **Индивидуальная поддержка** — студент, не проявляющий активность, может нуждаться в дополнительном внимании;
- **Оценка эффективности** — сравнение результатов до и после внедрения ИИ-материалов позволяет оценить их влияние на усвоение;
- **Адаптация темпа** — аналитика по времени изучения помогает оценить, насколько объёмный материал для студентов.

## 7. Интеграция в учебный процесс

### 7.1. Возможные сценарии использования

| Сценарий | Описание | Рекомендуемый режим |
|---|---|---|
| **Подготовка к лекции** | Студенты изучают конспект ИИ-тьютора перед лекцией для получения базового представления темы | Базовый уровень, краткий конспект |
| **Закрепление после лекции** | Студенты проходят тест ИИ-тьютора после лекции для проверки понимания | Средний уровень, 10–15 вопросов |
| **Самостоятельная работа** | Студенты изучают материал и выполняют задания в рамках самоподготовки | Продвинутый уровень, пояснения включены |

| **Подготовка к экзамену** | Генерация повторительных материалов и тестов по всему курсу | Все уровни, максимальное количество вопросов |
| **Работа с отстающими** | Создание упрощённых материалов для студентов, испытывающих затруднения | Базовый уровень, краткие пояснения |

### 7.2. Пример интеграции в план занятия

**Тема:** «Датчики температуры в промышленной автоматизации»

| Этап занятия | Время | Деятельность преподавателя | Деятельность студентов |
|---|---|---|---|
| 1. Подготовка (домашнее задание) | 30 мин (дом.) | Заблаговременно загрузить лекцию, сгенерировать конспект и опубликовать | Изучение конспекта ИИ-тьютора |
| 2. Начало занятия | 5 мин | Краткий опрос по материалам конспекта | Ответы на вопросы преподавателя |
| 3. Основная часть (лекция) | 40 мин | Изложение материала с акцентом на практические аспекты и демонстрацию оборудования | Конспектирование, вопросы |
| 4. Закрепление | 15 мин | Пояснения к сложным вопросам, разбор ошибок | Проходят тест ИИ-тьютора (10 вопросов) |
| 5. Итоги | 5 мин | Обсуждение результатов тестирования, рекомендации | Анализ собственных ошибок |

### 7.3. Рекомендации по планированию

1. **Загружайте материалы заранее** — минимум за 2–3 дня до занятия, чтобы успеть проверить и опубликовать;
2. **Не ограничивайтесь одним форматом** — комбинируйте конспекты, тесты и пояснения;
3. **Обновляйте материалы** — при изменении лекции обновляйте и сгенерированные материалы;
4. **Согласовывайте с рабочей программой** — проверяйте соответствие тем ИИ-материалов рабочей программе дисциплины;
5. **Используйте аналитику** — регулярно (раз в 2 недели) просматривайте отчёты и корректируйте подход.

## 8. Методические рекомендации

Ниже представлены рекомендации, основанные на педагогическом опыте и лучших практиках использования ИИ в образовании.

### Рекомендация 1. Начинайте с малого

Не пытайтесь сразу сгенерировать материалы на весь семестр. Начните с одной-двух тем, проверьте качество, оцените, как студенты воспринимают формат, и постепенно расширяйте использование. Это позволит выявить и устранить проблемы до масштабирования.

### Рекомендация 2. Всегда проверяйте результат

ИИ-модель, как бы она ни была настроена, может допускать ошибки. Каждый сгенерированный конспект и каждый тест должны быть лично проверены преподавателем-экспертом. Время, затраченное на проверку, значительно меньше времени, необходимого для создания материалов вручную, но пренебрегать им нельзя.

### Рекомендация 3. Используйте ИИ для дифференциации

Сгенерируйте материалы трёх уровней сложности по одной и той же теме: базовый (для слабых студентов), средний (для основной группы) и продвинутый (для сильных студентов и тех, кто готовится к олимпиадам). Это позволит каждому студенту работать в своём темпе и на своём уровне.

### Рекомендация 4. Комбинируйте ИИ-материалы с традиционными методами

ИИ-тьютор отлично работает в связке с практическими занятиями, лабораторными работами и демонстрациями оборудования. Например: конспект ИИ — для теории, тест ИИ — для самопроверки, лабораторная работа — для практического закрепления навыков работы с реальными датчиками и контроллерами.

### Рекомендация 5. Объясните студентам принцип работы системы

Расскажите студентам, что материалы подготовлены с помощью ИИ и могут содержать неточности. Поощряйте критическое отношение к сгенерированному контенту и просьбы сообщать о найденных ошибках. Это развивает навыки критического мышления и профессиональной бдительности — важные качества для будущих специалистов по автоматизации.

### Рекомендация 6. Не полагайтесь только на ИИ-тесты

Тесты ИИ-тьютора хорошо подходят для самопроверки и текущего контроля, но они не должны заменять полноценные контрольные работы и экзамены. Используйте ИИ-тесты как один из инструментов в системе оценки, дополняя их традиционными формами контроля.

### Рекомендация 7. Вносите специфику предприятия

ИИ обучается на общих данных и может не знать специфики конкретных предприятий, на которых проходят практику ваши студенты. Дополняйте сгенерированные материалы примерами и задачами, основанными на реальном опыте партнёрских предприятий.

### Рекомендация 8. Храните архив версий

Сохраняйте предыдущие версии сгенерированных материалов. Это позволит:
- Отслеживать эволюцию содержания;
- Восстановить материал, если новая версия оказалась хуже;
- Анализировать, какие формулировки работали лучше с точки зрения результатов тестирования.

### Рекомендация 9. Работайте в команде

Если по одной дисциплине работают несколько преподавателей, согласуйте подход к использованию ИИ-тьютора: общие требования к качеству, единые критерии проверки, распределение тем для генерации. Это обеспечит единство образовательного пространства.

### Рекомендация 10. Собирайте обратную связь

Регулярно опрашивайте студентов о качестве ИИ-материалов. Используйте как формальные опросы в Moodle, так и неформальные обсуждения на занятиях. Обратная связь от студентов — ценнейший источник информации для улучшения процесса.

## 9. Ограничения системы

Важно понимать границы возможностей ИИ-тьютора, чтобы не создавать ложных ожиданий и использовать систему там, где она действительно эффективна.

### 9.1. Что ИИ-тьютор НЕ умеет

| Ограничение | Пояснение |
|---|---|
| **Работать с графикой и чертежами** | Система анализирует только текст; принципиальные схемы, электрические чертежи, изображения оборудования не обрабатываются |
| **Проверять лабораторные работы** | Оценка практических навыков работы с оборудованием возможна только преподавателем |
| **Заменять живое общение** | ИИ не может вести диалог со студентом, учитывать контекст его предыдущих вопросов, мимику и уровень понимания в реальном времени |
| **Гарантировать отсутствие ошибок** | ИИ-модель может генерировать правдоподобные, но неверные утверждения (галлюцинации) |
| **Работать с материалами других форматов** | Система не обрабатывает видео, аудио, презентации PowerPoint, интерактивные симуляции |
| **Учитывать специфику конкретного предприятия** | Без дополнительного контекста ИИ не может адаптировать материал под конкретное производство |
| **Оценивать практические компетенции** | Навыки монтажа, наладки и эксплуатации средств автоматизации проверяются только в ходе практических занятий |
| **Заменять учебник** | ИИ-материалы являются дополнением к учебной литературе, а не её заменой |

### 9.2. Когда лучше создавать материалы вручную

- **Уникальный контент** — авторские методические разработки, оригинальные задачи на основе опыта предприятия;
- **Высокостандартизированные тесты** — контрольные работы, используемые для итоговой аттестации, где каждый вопрос прошёл многолетнюю валидацию;
- **Материалы с объёмными таблицами и чертежами** — информация, которая не может быть адекватно представлена в текстовом формате;
- **Междисциплинарные материалы** — темы на стыке нескольких дисциплин, требующие глубокого понимания связей, которые ИИ может не уловить.

## 10. Устранение неполадок

### 10.1. Частые проблемы и решения

| № | Проблема | Возможная причина | Решение |
|---|---|---|---|
| 1 | При нажатии «Сгенерировать» ничего не происходит | Браузер заблокировал запрос; медленное соединение с сервером | Обновите страницу (F5); проверьте подключение к локальной сети; попробуйте другой браузер |
| 2 | Система выдаёт ошибку «Файл слишком большой» | Объём файла превышает ограничение (10 000 символов) | Разделите материал на две части и загрузите последовательно |
| 3 | Система выдаёт ошибку «Неподдерживаемый формат» | Загружен файл не в формате .txt | Сохраните файл в формате «Обычный текст (.txt)» с кодировкой UTF-8 |
| 4 | Сгенерированный текст содержит «иероглифы» или искажённые символы | Неверная кодировка файла | Пересохраните файл в кодировке UTF-8; при сохранении из Word выберите «UTF-8» в параметрах |
| 5 | ИИ сгенерировал неверный ответ в тесте | Галлюцинация модели; нечёткая формулировка в исходном тексте | Отредактируйте вопрос вручную; проверьте исходный материал |
| 6 | Генерация занимает очень долго (более 2 минут) | Нагрузка на сервер; слишком большой объём текста | Подождите или повторите попытку позже; уменьшите объём загружаемого материала |
| 7 | Не отображается аналитика | Данные ещё не накоплены; недостаточно обращений студентов | Подождите накопления данных (минимум 5–10 обращений студентов) |
| 8 | Не могу найти раздел «ИИ-тьютор» в курсе | Недостаточные права доступа; модуль не активирован | Обратитесь к администратору Moodle для предоставления прав или активации модуля |
| 9 | При публикации материала студенты его не видят | Материал сохранён как черновик; ограничены права доступа | Проверьте статус публикации и права доступа для роли «Студент» |
| 10 | ИИ-тьютор не отвечает (сервер недоступен) | Сервер перезагружается или неисправен | Обратитесь к системному администратору колледжа |

### 10.2. Куда обращаться за помощью

| Тип проблемы | Контакты |
|---|---|
| Технические проблемы (сервер, доступ, ошибки) | Системный администратор колледжа, каб. ___ |
| Методические вопросы (качество материалов, рекомендации) | Методический кабинет, каб. ___ |
| Проблемы с Moodle (настройки курса, права доступа) | Администратор Moodle, каб. ___ |
| Вопросы по дисциплине (содержание, корректность) | Председатель цикловой комиссии технических дисциплин |

## 11. Часто задаваемые вопросы

### В1. Нужно ли мне иметь специальные знания в области ИИ для работы с системой?

**Нет.** ИИ-тьютор спроектирован так, чтобы быть доступным для любого преподавателя без технической подготовки. Все функции выполняются через интуитивно понятный интерфейс Moodle. Вы загружаете текст — система возвращает результат. Единственное, что требуется — базовые навыки работы с компьютером и Moodle.

### В2. Могут ли студенты использовать ИИ-тьютор напрямую для генерации материалов?

**Нет.** Функция генерации доступна только преподавателям. Студенты имеют доступ только к опубликованным материалам (конспектам, тестам, пояснениям). Это обеспечивает контроль качества — все материалы проверяются преподавателем перед публикацией.

### В3. Как обеспечить конфиденциальность учебных материалов?

Все материалы хранятся на **локальном сервере колледжа** и не передаются во внешние сервисы. ИИ-модель также развёрнута локально. Доступ к материалам имеет только персонал колледжа, авторизованный в системе Moodle. Система не подключена к интернету.

### В4. Что делать, если ИИ-тьютор сгенерировал материал низкого качества?

Повторите генерацию, предварительно уточнив исходный текст. Часто улучшение качества исходного материала (добавление заголовков, чётких определений, структурирование) существенно повышает качество результата. Если результат всё равно неудовлетворительный — создайте материал вручную и используйте ИИ-тьютор для других тем.

### В5. Можно ли использовать ИИ-тьютор для других дисциплин?

Технически система поддерживает загрузку текстов по любой дисциплине, однако ИИ-модель была предварительно настроена на тематику промышленной автоматики и измерительных систем. Для других дисциплин качество генерации может быть ниже. Обратитесь к системному администратору, если вы хотите расширить предметную область системы.

### В6. Как часто рекомендуется обновлять сгенерированные материалы?

Рекомендуется обновлять материалы при каждом значительном изменении лекционного содержания, но не реже одного раза в учебный год. Также обновление целесообразно при появлении новых стандартов (ГОСТ), изменений в нормативной документации или обновлении оборудования в лабораториях.

### В7. Считается ли использование ИИ-тьютора плагиатом?

**Нет.** ИИ-тьютор не копирует существующие тексты из интернета, а генерирует оригинальные материалы на основе загруженного вами содержания. Вы являетесь автором исходного материала и несёте ответственность за итоговое содержание. С точки зрения академической этики, использование ИИ-тьютора аналогично использованию любого вспомогательного инструмента (текстового редактора, шаблона теста).

### В8. Сколько времени занимает проверка сгенерированных материалов?

В среднем — **10–20 минут** на конспект и **15–30 минут** на тест из 10–15 вопросов. Это значительно меньше, чем создание аналогичных материалов вручную (1–3 часа на конспект, 2–4 часа на тест). По мере привыкания к системе и пониманию её особенностей время проверки сокращается.

### В9. Можно ли редактировать сгенерированный тест после того, как студенты уже начали его проходить?

**Да, но с ограничениями.** Вопросы, на которые уже были даны ответы студентами, нельзя изменить или удалить — это необходимо для сохранения корректности статистики. Вы можете добавлять новые вопросы и редактировать те, на которые ещё никто не ответил. Рекомендуется завершить проверку теста **до** его публикации.

### В10. Как обеспечить равный доступ студентов к ИИ-материалам, если у некоторых нет компьютера дома?

На территории колледжа организованы компьютерные классы и зона свободного доступа к Wi-Fi. Все ИИ-материалы доступны через Moodle, который открывается в любом браузере, в том числе на мобильных устройствах. Рекомендуется также выделять время на самостоятельную работу с ИИ-материалами в расписании занятий (в компьютерном классе или читальном зале).
