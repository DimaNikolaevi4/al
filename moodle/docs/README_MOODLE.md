# ИИ-Тьютор СПО — Плагин для Moodle

## Описание

Плагин **ИИ-Тьютор СПО** интегрирует систему искусственного интеллекта в LMS Moodle для системы среднего профессионального образования (СПО). Специальность 15.02.14 «Оснащение средствами автоматизации технологических процессов и производств».

### Возможности

- 📝 **Генерация конспекта** — автоматическое создание структурированного конспекта по тексту лекции
- 📋 **Генерация теста** — создание тестовых вопросов по материалам лекции с настройкой сложности
- 💬 **Чат с ИИ** — диалог с ИИ-тьютором по учебным материалам
- ⚡ **Асинхронная обработка** — все запросы обрабатываются асинхронно через систему задач
- 🔐 **Ролевая модель** — гибкое управление правами доступа через роли Moodle

## Установка

### 1. Копирование файлов

Скопируйте содержимое директории `local/aitutor/` в директорию `local/` вашего Moodle-сервера:

```bash
cp -r local/aitutor/ /path/to/moodle/local/aitutor/
```

Структура файлов после установки:

```
moodle/local/aitutor/
├── version.php              # Версия плагина
├── settings.php             # Страница настроек
├── ajax.php                 # AJAX-эндпоинт для блока
├── classes/
│   └── service.php          # Сервис взаимодействия с API
├── db/
│   ├── install.xml          # Определение таблиц БД
│   ├── install.php          # Скрипт первичной установки
│   └── access.php           # Определение прав доступа
├── lang/
│   ├── ru/local_aitutor.php # Русские строки
│   └── en/local_aitutor.php # Английские строки
├── amd/
│   └── src/
│       └── aitutor_block.js # JavaScript-модуль блока
└── block_aitutor/           # Блок для боковой панели
    ├── block_aitutor.php
    ├── version.php
    ├── edit_form.php
    ├── styles.css
    └── lang/
        ├── ru/block_aitutor.php
        └── en/block_aitutor.php
```

### 2. Установка через веб-интерфейс

1. Зайдите на сайт Moodle как администратор
2. Перейдите в **Администрирование сайта → Плагины → Установка**
3. Нажмите **«Обновить список плагинов»** — Moodle обнаружит новый плагин
4. Нажмите **«Установить плагин»** для `local_aitutor`
5. После установки появится блок `ИИ-Тьютор СПО` в списке доступных блоков

### 3. Добавление блока на страницу курса

1. Откройте нужный курс
2. Включите режим редактирования
3. В меню **«Добавить блок»** выберите **«ИИ-Тьютор СПО»**
4. Блок появится в боковой панели курса

## Настройка

### Глобальные настройки плагина

Перейдите: **Администрирование сайта → Плагины → Локальные плагины → ИИ-Тьютор СПО**

| Параметр | Описание | Значение по умолчанию |
|----------|----------|-----------------------|
| URL сервера API | Базовый адрес FastAPI-сервера | `http://localhost:8000` |
| API-ключ | Ключ авторизации (если требуется) | пусто |
| Таймаут запросов | Максимальное время ожидания (сек.) | `120` |
| Включить генерацию конспекта | Показать кнопку конспекта | Да |
| Включить генерацию тестов | Показать кнопку тестов | Да |
| Включить чат с ИИ | Показать кнопку чата | Да |
| Макс. длина лекции | Ограничение символов | `50000` |
| Сложность тестов | Уровень по умолчанию | `Средний` |

### Настройка блока (на уровне курса)

При редактировании блока можно выбрать, какие кнопки отображать:
- Кнопка «Создать конспект»
- Кнопка «Сгенерировать тест»
- Кнопка «Спросить ИИ»

### Управление правами доступа

Права назначаются через стандартный механизм Moodle:

| Право | Назначение | Роль по умолчанию |
|-------|------------|-------------------|
| `local/aitutor:use` | Использование функций ИИ-Тьютора | Студент, Преподаватель |
| `local/aitutor:manage` | Управление настройками плагина | Менеджер |

## Использование

### Для студентов

1. Перейдите на страницу курса
2. В боковой панели найдите блок **«ИИ-Тьютор СПО»**
3. Нажмите нужную кнопку:
   - **«Создать конспект»** — вставьте текст лекции, нажмите «Отправить», дождитесь результата
   - **«Сгенерировать тест»** — вставьте текст лекции, выберите сложность, получите тест
   - **«Спросить ИИ»** — задайте вопрос по материалу курса

### Для преподавателей

Преподаватели имеют полный доступ ко всем функциям. Дополнительно они могут:
- Настраивать доступные кнопки в блоке
- Управлять правами доступа через назначения ролей

## Решение проблем

### Сервер API недоступен

**Симптом:** Сообщение «Сервер ИИ-Тьютора недоступен»

**Решение:**
1. Проверьте, запущен ли API-сервер: `curl http://localhost:8000/api/v1/health`
2. Проверьте настройки URL в конфигурации плагина
3. Проверьте сетевую доступность между Moodle и API-сервером
4. Проверьте логи Moodle: `tail -f /path/to/moodle/data/moodle.log | grep local_aitutor`

### Таймаут запросов

**Симптом:** Запрос обрабатывается слишком долго

**Решение:**
1. Увеличьте таймаут в настройках плагина (до 300 секунд)
2. Проверьте нагрузку на API-сервер
3. Уменьшите объём отправляемого текста лекции

### Пустой результат

**Симптом:** Статус «Готово», но результат пустой

**Решение:**
1. Проверьте логи API-сервера
2. Убедитесь, что модель ИИ корректно сконфигурирована
3. Попробуйте с более коротким текстом

### Блок не отображается

**Решение:**
1. Убедитесь, что плагин установлен: **Администрирование → Плагины → Обзор**
2. Проверьте права доступа: пользователь должен иметь `local/aitutor:use`
3. Очистите кэш Moodle: **Администрирование → Обслуживание сайта → Очистить кэш**

### Поток данных

1. **Студент** нажимает кнопку в блоке → открывается модальное окно
2. **JavaScript** отправляет AJAX-запрос на `local/aitutor/ajax.php`
3. **PHP-обработчик** проверяет права, вызывает `ai_tutor_service`
4. **Сервис** отправляет HTTP-запрос к FastAPI через Moodle `curl`
5. **API** создаёт задачу и возвращает `task_id`
6. **PHP** сохраняет запрос в БД `local_aitutor_requests`
7. **JavaScript** начинает опрос статуса задачи (каждые 3 сек)
8. При получении результата — отображается в модальном окне

## Структура базы данных

### `local_aitutor_requests`

Хранит историю всех запросов пользователей.

| Поле           | Тип      | Описание                           |
|----------------|----------|------------------------------------|
| id             | INT      | Первичный ключ                     |
| userid         | INT      | ID пользователя                    |
| courseid       | INT      | ID курса                           |
| request_type   | CHAR(20) | Тип: summary, quiz, chat           |
| input_text     | TEXT     | Входной текст                      |
| task_id        | CHAR     | ID асинхронной задачи              |
| status         | CHAR(20) | Статус: pending, processing, complete, failed |
| result_text    | TEXT     | Результат в формате JSON           |
| error_message  | TEXT     | Сообщение об ошибке                |
| timecreated    | INT      | Время создания                     |
| timemodified   | INT      | Время последнего изменения         |

### `local_aitutor_settings_log`

Аудит-лог изменений настроек плагина.

| Поле         | Тип      | Описание                  |
|--------------|----------|---------------------------|
| id           | INT      | Первичный ключ            |
| userid       | INT      | ID администратора         |
| setting_name | CHAR     | Название настройки        |
| old_value    | TEXT     | Предыдущее значение       |
| new_value    | TEXT     | Новое значение            |
| timestamp    | INT      | Время изменения           |
