#### Результат 1.1.4 — Формат датасета

Определено на основе: tutor.py (метод `chat()` использует `apply_chat_template()`), библиотеки Transformers/Hugging Face (стандарт SFTTrainer), чат-шаблона Mistral Small 3.1.

**Формат файла:** JSONL (один JSON-объект на строку, кодировка UTF-8 без BOM)

**Структура каждой строки:**

```json
{
  "messages": [
    {"role": "system", "content": "<системный промпт для типа задачи>"},
    {"role": "user", "content": "<входные данные (текст лекции, вопрос)>"},
    {"role": "assistant", "content": "<ожидаемый ответ модели>"}
  ],
  "metadata": {
    "task_type": "<тип задачи: summary_basic, quiz_multiple_choice и т.д.>",
    "subject": "<предмет: industrial_auto, measurement_systems и т.д.>",
    "difficulty": "<сложность: basic, intermediate, advanced>",
    "topic": "<тема занятия>",
    "source": "<источник: lecture, textbook, moodle>"
  }
}
```

**Обязательные поля:** `messages` (массив из 3 элементов: system → user → assistant)

**Опциональные поля:** `metadata` (для аналитики и балансировки, не используется при обучении)

**Соответствие методам tutor.py:**

| Поле messages | Метод tutor.py | Роль |
|---|---|---|
| `system` | `_build_summary_prompt()` / промпт в `generate_quiz()` | Инструкция модели: что делать |
| `user` | `lecture_text` аргумент методов | Текст лекции / вопрос студента |
| `assistant` | Return value методов | Ожидаемый результат генерации |

**Пример строки для T1 (конспект базовый):**

```json
{
  "messages": [
    {"role": "system", "content": "Ты — ИИ-тьютор для студентов техникума. Сделай краткий структурированный конспект лекции. Выдели основные тезисы, используй маркированные списки."},
    {"role": "user", "content": "Тема: Основы промышленной автоматизации. Промышленная автоматизация — это применение систем управления и информационных технологий для управления промышленными процессами. Основные компоненты: ПЛК, датчики, исполнительные механизмы, SCADA-системы."},
    {"role": "assistant", "content": "## Конспект: Основы промышленной автоматизации\n\n**Промышленная автоматизация** — управление технологическими процессами с помощью систем управления и ИТ.\n\n### Основные компоненты:\n- **ПЛК** (программируемый логический контроллер) — «мозг» автоматики, выполняет программу управления\n- **Датчики** — измеряют параметры процесса (температура, давление, расход)\n- **Исполнительные механизмы** — выполняют команды контроллера (клапаны, моторы)\n- **SCADA-система** — интерфейс оператора для мониторинга и управления"}
  ],
  "metadata": {
    "task_type": "summary_basic",
    "subject": "industrial_auto",
    "difficulty": "basic",
    "topic": "Основы промышленной автоматизации",
    "source": "lecture"
  }
}
```

**Примечание по 1.1.5 (🔶):** Формат `messages` с ролями `system/user/assistant` совместим с `tokenizer.apply_chat_template()` из tutor.py, что решает задачу 1.1.5. Для применения достаточно: `tokenizer.apply_chat_template(row["messages"], tokenize=False)`.

**1. Методика оценки качества источников**

Каждый источник (учебник, методическое пособие, конспект лекции, тест) оценивается по 6 критериям по шкале 1–5 баллов:

| # | Критерий | Описание | Вес |
|---|----------|----------|:---:|
| K1 | Актуальность | Год издания ≤5 лет (5 баллов), ≤10 лет (4 балла), >10 лет (2 балла). Обновлённые издания — +1 балл. | 20% |
| K2 | Соответствие ФГОС СПО | Наличие в УМК дисциплины (ФГР/ФОП). Покрытие mandatory-компетенций из ФГОС 15.02.14. | 25% |
| K3 | Полнота покрытия темы | Соответствие рабочей программе дисциплины (РПД). Наличие теории, примеров, задач, иллюстраций. | 20% |
| K4 | Доступность изложения | Уровень языка соответствует СПО (без излишнего академизма). Наличие примеров, определений, схем. | 15% |
| K5 | Структурированность | Наличие оглавления, глоссария, контрольных вопросов. Логическая последовательность изложения. | 10% |
| K6 | Цифровая доступность | Наличие электронного формата (PDF/Word). Пригодность для оцифровки (OCR-качество сканов ≥95%). | 10% |

**Формула итоговой оценки:** O = K1×0.20 + K2×0.25 + K3×0.20 + K4×0.15 + K5×0.10 + K6×0.10

**Порог включения в датасет:** O ≥ 3.5 баллов (источники ниже порога — только как вспомогательные, не для основного датасета).

**2. Критерии качества тестов для включения в датасет**

Каждый тест/вопрос из Moodle оценивается по 5 критериям (шкала 1–5):

| # | Критерий | Описание | Вес |
|---|----------|----------|:---:|
| C1 | Техническая корректность | Валидность разметки в Moodle (правильный ответ указан, нет конфликтов вариантов, все поля заполнены). | 20% |
| C2 | Содержательная точность | Фактологическая и терминологическая правильность. Соответствие актуальным стандартам (ГОСТ, ПУЭ, учебники из Результата 2.1.2). | 30% |
| C3 | Соответствие уровню СПО | Сложность соответствует профилю студента СИТ (Результат 1.1.3: 2–3 курс, базовые знания физики и информатики). Не слишком простой, не требует высшей математики. | 20% |
| C4 | Наличие обратной связи | Объяснение правильного/неправильного ответа (Moodle: поле «Общая обратная связь»). Ценно для датасета — модель учится не только отвечать, но и объяснять. | 15% |
| C5 | Тематическая привязка | Вопрос привязан к конкретной теме и дисциплине (S1–S4). Не «общий вопрос без контекста». | 15% |

**Формула:** Q = C1×0.20 + C2×0.30 + C3×0.20 + C4×0.15 + C5×0.15

**Порог включения:** Q ≥ 3.5 баллов. Вопросы с Q < 3.5 — отправляются на доработку преподавателю или исключаются.
