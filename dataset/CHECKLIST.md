# Чеклист — Дорожная карта создания датасета для дообучения ИИ-тьютора СПО

---

## 1. Подготовительный этап: планирование и целеполагание

- [ ] Определить целевые задачи дообучения: конспекты, тесты, объяснения, адаптация материала
- [ ] Предметные области для генерации данных: дисциплины СПО, специальности, учебные планы
- [ ] Анализ целевой аудитории студентов СПО (уровень подготовки, возраст, специфика)
- [ ] Согласовать формат конечного промпта (input-output пары для модели)
- [ ] Изучить формат данных, который принимает модель Mistral Small 3.1 (инструкции, чат-формат)

---

## 2. Сбор исходных данных: источники и их оценка

- [ ] Выбор источников данных
    - [ ] Учебники, методические пособия и конспекты лекций преподавателей СИТ
    - [ ] Существующие тесты и задания для самопроверки из Moodle
    - [ ] Нормативные документы: ФГОС СПО, рабочие программы, учебные планы
    - [ ] Дополнительная литература и справочные материалы по специальности
- [ ] Оценка источников
    - [ ] Проверка качества и достоверности каждого учебного материала
    - [ ] Исключение устаревших данных и материалов с ошибками
    - [ ] Оценка разнообразия тем и_coverage предметных областей
    - [ ] Проверка орфографии, пунктуации и стилистической однородности текстов

---

## 3. Разработка инструкций (system prompts)

- [ ] Составление перечня типов задач для тьютора
- [ ] Написание базового system prompt для каждой задачи
- [ ] Создание примеров входных данных (prompts) и ожидаемых ответов
- [ ] Валидация корректности каждого prompt-response на предмет соответствия образовательным стандартам

---

## 4. Форматирование данных

- [ ] Выбор формата датасета JSONL (system-message, assistant, instruction)
- [ ] Подготовить шаблон структуры датасета (например: byte-pair (input, output))
- [ ] Проверить соответствие данных формату, который использует библиотека Transformers
- [ ] Удалить дубликаты и перекрывающиеся записи
- [ ] Разметить данные по категориям и предметам (для управления балансом классов)

---

## 5. Типизация данных

- [ ] Определить и описать типы контента, которые будет генерировать тьютор
- [ ] Создать классификацию по типам задач (конспект, тест, объяснение, адаптация, перевод)
- [ ] Нормализация стилей изложения для каждого типа контента
- [ ] Обеспечение единообразия в формате строк, чисел, special symbols
- [ ] Нормализация текстов на предмет соответствия образовательным стандартам (терминология, ГОСТы)

> *Опционально:* Добавить метаданные для каждого примера — предмет, уровень сложности, целевая специальность, что улучшит управление балансом датасета при обучении.

---

## 6. Чистка данных

- [ ] Удаление шума и нерелевантной информации из текстов
- [ ] Удаление повреждённых примеров input-output пар с неполными данными
- [ ] Удаление дублирующихся или почти идентичных записей
- [ ] Проверка минимальной длины текста (не короче N символов/токенов)
- [ ] Удаление примеров с Персональными данными студентов (ФИО, группы, оценки)
- [ ] Проверка на соответствие цензурным требованиям и этическим нормам
- [ ] Автоматизация базовой проверки оборотов (автоматический скрипт)

---

## 7. Балансировка датасета

- [ ] Проверить равномерность распределения данных по предметам
- [ ] Проверить и отрегулировать соотношение примеров по типам задач (конспекты, тесты, объяснения)
- [ ] Проверить баланс по уровню сложности заданий (базовый, углублённый, продвинутый)
- [ ] Разделение датасета (80% — обучающая выборка, 20% — валидационная) для базовой оценки

---

## 8. Чистовое ревью датасета

- [ ] Аудит повторов модели на разноборчивые паттерны
    - [ ] Проверка на повторяющиеся шаблоны ответов
    - [ ] Наличие повторяющихся фраз
    - [ ] Проверка качества и разнообразия генерации
    - [ ] Анализ полиномиальности шаблонов ответов и обучающих данных
    - [ ] Анализ токенизации данных
- [ ] Аудит метрик качества и разнообразия
    - [ ] Проверить длину входных и выходных текстов в выборках
    - [ ] Анализ распределения классов и категорий в датасете
- [ ] Итоговая проверка экспертами (преподаватели СИТ)
