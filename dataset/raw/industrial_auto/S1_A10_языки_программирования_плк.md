# Лекция 10. Языки программирования ПЛК

## Стандарт IEC 61131-3

До конца 1990-х годов каждый производитель ПЛК использовал собственный язык программирования. Программа, написанная для ПЛК Siemens, не работала на ПЛК Allen-Bradley. Это создавало серьёзные проблемы: инженеру приходилось изучать новый язык при переходе на оборудование другого производителя.

Для решения этой проблемы Международная электротехническая комиссия (IEC) разработала стандарт **IEC 61131-3**, который определяет пять стандартизированных языков программирования ПЛК:

1. **LD** (Ladder Diagram) — релейные схемы (лестничные диаграммы)
2. **FBD** (Function Block Diagram) — функциональные блоковые диаграммы
3. **ST** (Structured Text) — структурированный текст
4. **IL** (Instruction List) — список инструкций
5. **SFC** (Sequential Function Chart) — последовательностные функциональные схемы

Все современные ПЛК (Siemens, ОВЕН, Schneider Electric, Allen-Bradley) поддерживают эти языки, хотя названия могут немного отличаться.

Важно понимать, что в одном проекте можно **комбинировать** разные языки: например, основная логика — на FBD, а сложные вычисления — на ST.

## LD — Ladder Diagram (релейные схемы)

### Описание

**LD** (Ladder Diagram) — это графический язык, визуально похожий на **схему электрических соединений реле**. Программа на LD выглядит как две вертикальные шины (левая — «фаза», правая — «ноль»), между которыми расположены горизонтальные «рунги» (rungs) — строки логики.

Это самый старый и наиболее распространённый язык программирования ПЛК, особенно в Северной Америке. Он был создан для того, чтобы электрики и инженеры по КИПиА, привыкшие к чтению релейных схем, могли легко освоить программирование ПЛК.

### Основные элементы

- **Контакт (Contact):** обозначает условие. Бывает нормально разомкнутым (NO — замыкается при сигнале) и нормально замкнутым (NC — размыкается при сигнале). Аналог кнопки или концевого выключателя.
- **Катушка (Coil):** обозначает действие. При прохождении тока через рунгу катушка «срабатывает» и устанавливает выходной сигнал. Аналог реле.
- **Блок таймера (TON, TOF, TP):** выполняет функции задержки.
- **Блок счётчика (CTU, CTD):** считает импульсы.

### Пример: Управление насосом с кнопками

Задача: насос включается кнопкой «Пуск» и отключается кнопкой «Стоп». При аварии (сигнал от датчика) насос должен немедленно отключиться.

```
|  Пуск          Стоп          Авария          |
|---| |-----------|/|------------|/|----------( )---|  (Насос)
|
|  Насос                                          |
|---| |-------------------------------| 1 сек TON |---|
|                                       Q          |
|                                       (Насос_зад)|
```

**Объяснение:**

- **Пуск (NO)** — кнопка «Пуск», нормально разомкнутый контакт. Нажали — контакт замкнулся.
- **Стоп (NC)** — кнопка «Стоп», нормально замкнутый контакт. Нажали — контакт разомкнулся.
- **Авария (NC)** — сигнал от датчика аварии, нормально замкнутый контакт. При аварии сигнал появляется, контакт размыкается, и насос отключается.
- **(Насос)** — катушка, управляющая выходом ПЛК, к которому подключён контактор насоса.

### Достоинства и недостатки LD

**Достоинства:**
- визуальная наглядность для специалистов по электрооборудованию;
- простота программирования простых логических задач;
- мгновенная «читаемость» — по схеме сразу видно логику работы.

**Недостатки:**
- сложно программировать сложные математические вычисления и алгоритмы;
- при большом количестве рунг программа становится трудночитаемой;
- неудобно для работы с массивами данных и строками.

**Когда использовать:** простые задачи логического управления (вкл/выкл, блокировки, таймеры), небольшие проекты.

## FBD — Function Block Diagram (функциональные блоки)

### Описание

**FBD** — это графический язык, в котором программа состоит из **функциональных блоков**, соединённых линиями (проводами) передачи данных. Каждый функциональный блок выполняет определённую операцию: математическую, логическую, PID-регулирование и т.д.

Входы блоков расположены слева, выходы — справа. Данные передаются от выхода одного блока ко входу другого по линиям связи.

### Основные функциональные блоки

- **Логические:** AND (И), OR (ИЛИ), NOT (НЕ), XOR (исключающее ИЛИ).
- **Математические:** ADD (сложение), SUB (вычитание), MUL (умножение), DIV (деление).
- **Сравнения:** GT (>), GE (>=), LT (<), LE (<=), EQ (=), NE (<>).
- **Таймеры:** TON (задержка включения), TOF (задержка отключения), TP (импульсный таймер).
- **Счётчики:** CTU (счётчик вперёд), CTD (счётчик назад).
- **PID:** блок ПИД-регулирования.
- **Аналоговый ввод/вывод:** SCALE (масштабирование сигнала), LIMIT (ограничение).

### Пример: Масштабирование аналогового сигнала

Задача: преобразовать сигнал с аналогового входа (0–32767) в физическое значение давления (0–10 бар).

```
  AI_raw (0..32767)        10.0
       |                    |
       v                    v
  +-----------+       +-----------+
  |   SCALE   |       |           |
  | IN=       |-------| IN_HI=   |
  | OUT=      |       +-----------+
  | IN_LO= 0  |
  | IN_HI=32767     0.0
  | OUT_LO= 0 |     |
  | OUT_HI= 10|-----|
  +-----------+     v
       |        +-----------+
       |        |           |
       |        | OUT_LO= 0 |
       |        +-----------+
       |             |
       v             v
  +-----------+
  |  P_bar    |  --> Результат: давление в барах
  +-----------+
```

На практике блок SCALE (или линейная интерполяция) имеет входы IN (входное значение), IN_LO, IN_HI (нижний и верхний пределы входного сигнала) и OUT_LO, OUT_HI (нижний и верхний пределы выходного значения). Формула:

$$OUT = (IN - IN\_LO) \times \frac{OUT\_HI - OUT\_LO}{IN\_HI - IN\_LO} + OUT\_LO$$

### Пример: ПИД-регулирование температуры

```
  T_actual (фактическая температура)
       |
       v
  +-----------+     T_set (заданная температура)
  |    SUB    |<----|
  |           |
  | e = T_set - T_actual (ошибка регулирования)
  +-----+-----+
        |
        v
  +-----------+
  |    PID    |----> AO_valve (управление клапаном 4-20 мА)
  | Kp= 2.0   |
  | Ki= 0.5   |
  | Kd= 0.1   |
  +-----------+
```

ПИД-блок вычисляет управляющее воздействие на основе ошибки регулирования (разности между заданным и фактическим значением) и трёх коэффициентов: пропорционального (Kp), интегрального (Ki) и дифференциального (Kd).

### Достоинства и недостатки FBD

**Достоинства:**
- наглядность — логика работы видна визуально;
- удобен для аналоговой обработки сигналов и ПИД-регулирования;
- функциональные блоки можно повторно использовать (сохранять в библиотеке);
- проще отлаживать, чем текстовые программы (видно промежуточные значения на каждом проводе).

**Недостатки:**
- при большом количестве блоков схема становится запутанной;
- сложно реализовать сложные алгоритмы с ветвлениями и циклами;
- «провода» могут пересекаться, создавая визуальный хаос.

**Когда использовать:** задачи с аналоговыми сигналами, ПИД-регулирование, обработка данных от датчиков, преобразование сигналов.

## ST — Structured Text (структурированный текст)

### Описание

**ST** — это текстовый язык программирования, по синтаксису похожий на язык **Pascal** (а отчасти — на C). ST является наиболее мощным и гибким языком из стандарта IEC 61131-3 для реализации сложных алгоритмов.

### Основные конструкции

**Операторы присваивания:**
```
Насос := TRUE;        (* присвоить переменной Насос значение TRUE *)
Температура_зад := 85.0;  (* присвоить значение 85.0 *)
```

**Арифметические операции:**
```
Результат := (A + B) * C / D;
Среднее := (T1 + T2 + T3) / 3.0;
```

**Операторы сравнения:**
```
IF Температура > 100.0 THEN
    Авария := TRUE;
END_IF;
```

**Логические операции:**
```
Насос_ВКЛ := (Кнопка_Пуск AND NOT Кнопка_Стоп AND NOT Авария);
```

**Условный оператор IF:**
```
IF Уровень < 1.0 THEN
    Насос := TRUE;
    Сообщение := 'Насос включён - низкий уровень';
ELSIF Уровень > 2.5 THEN
    Насос := FALSE;
    Сообщение := 'Насос выключен - высокий уровень';
ELSE
    Сообщение := 'Нормальный уровень';
END_IF;
```

**Оператор цикла FOR:**
```
Сумма := 0;
FOR i := 1 TO 10 DO
    Сумма := Сумма + Массив[i];
END_FOR;
Среднее := Сумма / 10.0;
```

**Оператор цикла WHILE:**
```
WHILE Датчик_Готов = FALSE DO
    Ждать_1_сек();
END_WHILE;
```

**Оператор CASE:**
```
CASE Режим OF
    1:  (* Ручной *)
        Скорость := Ручная_Скорость;
    2:  (* Автоматический *)
        Скорость := Авто_Скорость;
    3:  (* Останов *)
        Скорость := 0.0;
ELSE
    Скорость := 0.0;
    Ошибка := TRUE;
END_CASE;
```

### Пример: Управление с гистерезисом

Задача: включить нагреватель при температуре ниже 20 °C и выключить при температуре выше 25 °C (гистерезис 5 °C для предотвращения частого включения/выключения).

```
IF Нагреватель_ВКЛ = FALSE THEN
    (* Нагреватель выключен — ждём падения температуры *)
    IF Температура < 20.0 THEN
        Нагреватель_ВКЛ := TRUE;
    END_IF;
ELSE
    (* Нагреватель включён — ждём повышения температуры *)
    IF Температура > 25.0 THEN
        Нагреватель_ВКЛ := FALSE;
    END_IF;
END_IF;
```

### Достоинства и недостатки ST

**Достоинства:**
- наивысшая выразительность и гибкость;
- удобен для сложных математических вычислений;
- легко реализовать циклы, массивы, строковые операции;
- компактный код (много логики в нескольких строках);
- familiarity для программистов (синтаксис похож на Pascal/C).

**Недостатки:**
- менее нагляден, чем графические языки (LD, FBD);
- требует навыков программирования;
- не всегда понятен для инженеров-электриков без опыта программирования;
- проще допустить логическую ошибку, которая не видна «на глаз».

**Когда использовать:** сложные математические вычисления, обработка массивов данных, сложная логика с ветвлениями и циклами, алгоритмы с условиями.

## IL — Instruction List (список инструкций)

### Описание

**IL** — это низкоуровневый текстовый язык, похожий на язык ассемблера. Каждая строка содержит одну инструкцию, которая оперирует с данными, хранящимися в стеке (аккумуляторе).

### Пример

```
LD Кнопка_Пуск    (* Загрузить значение Кнопка_Пуск в аккумулятор *)
AND NOT Кнопка_Стоп  (* Выполнить логическое И с инверсией Кнопка_Стоп *)
ST Насос            (* Сохранить результат в переменную Насос *)
```

### Статус

Стандарт IEC 61131-3 (3-е издание) **исключил IL из списка основных языков**, обозначив его как устаревший. Современные ПЛК и среды программирования (например, Codesys V3.5) больше не поддерживают IL или поддерживают в режиме совместимости.

В настоящее время IL практически не используется для новых проектов.

## SFC — Sequential Function Chart (графсет)

### Описание

**SFC** — это графический язык для описания **последовательных (шаговых) алгоритмов**. Программа на SFC выглядит как набор **шагов** (steps), соединённых **переходами** (transitions).

**Шаг** — это состояние системы, в котором выполняются определённые действия (например, «Насос включён», «Клапан открыт», «Ожидание 10 секунд»).

**Переход** — это условие перехода от одного шага к другому. Переход срабатывает, когда выполняется заданное логическое условие.

### Пример: Алгоритм автоматической промывки фильтра

```
  +-------+
  | Шаг 0 | Инициализация
  +-------+
      |
      | [Кнопка "Промывка" нажата]
      v
  +-------+
  | Шаг 1 | Закрыть клапан подачи, открыть клапан сброса
  +-------+
      |
      | [Клапан сброса открыт И таймер 2 сек истёк]
      v
  +-------+
  | Шаг 2 | Включить насос промывки
  +-------+
      |
      | [Насос включён И таймер 30 сек истёк]
      v
  +-------+
  | Шаг 3 | Отключить насос, закрыть клапан сброса, открыть клапан подачи
  +-------+
      |
      | [Клапан подачи открыт]
      v
  +-------+
  | Шаг 0 | Возврат в начальное состояние
  +-------+
```

### Достоинства и недостатки SFC

**Достоинства:**
- наглядность последовательных алгоритмов;
- удобство описания пошаговых технологических процессов;
- встроенные механизмы обработки ошибок и альтернативных ветвей (параллельные ветвления).

**Недостатки:**
- не подходит для задач непрерывного регулирования (ПИД-регулирование лучше делать на FBD или ST);
- при большом количестве шагов и ветвлений схема усложняется.

**Когда использовать:** пошаговые процессы (загрузка → обработка → выгрузка), автоматические режимы работы, стартовые и остановочные процедуры.

## Выбор языка для задачи

Выбор языка программирования зависит от характера задачи и квалификации разработчика:

| Задача | Рекомендуемый язык | Причина |
|---|---|---|
| Простая логика (вкл/выкл, блокировки) | **LD** | Наглядность, простота |
| ПИД-регулирование, обработка аналоговых сигналов | **FBD** | Наглядные блоки, удобная отладка |
| Сложные вычисления, математика, массивы | **ST** | Мощность и гибкость |
| Пошаговые алгоритмы, циклы работы установки | **SFC** | Наглядное описание шагов |
| Комбинированная задача | **FBD + ST** или **LD + ST** | Комбинация наглядности и мощности |

### Рекомендации для начинающих

1. Начните с **LD** для простых логических задач — это даст понимание базовых принципов.
2. Освойте **FBD** для работы с аналоговыми сигналами и ПИД-регулированием.
3. Изучите **ST** для задач, которые неудобно решать графически.
4. Познакомьтесь с **SFC** для пошаговых алгоритмов.
5. На реальных проектах комбинируйте языки для максимальной эффективности.

### Среды программирования

Для практической работы с языками IEC 61131-3 используются следующие среды:

- **CODESYS** (ранее 3S-Smart Software Solutions) — независимая среда, поддерживающая все пять языков IEC 61131-3. Используется многими производителями (ОВЕН, WAGO, Beckhoff, Danfoss и др.) как основа для своих систем программирования.
- **TIA Portal** (Siemens) — поддерживает LD, FBD, SFC, ST (в разных редакциях).
- **Кодекс-ВК / OWEN Logic** (ОВЕН) — среда для программирования ПЛК ОВЕН.
- **EcoStruxure Control Expert** (Schneider Electric) — поддерживает LD, FBD, ST, SFC, IL.
- **Studio 5000** (Rockwell Automation / Allen-Bradley) — поддерживает LD, FBD, ST, SFC.

## Резюме

- Стандарт **IEC 61131-3** определяет пять языков программирования ПЛК: LD, FBD, ST, IL и SFC.
- **LD** — графический язык, похожий на релейные схемы. Лучше всего подходит для простых логических задач.
- **FBD** — графический язык с функциональными блоками. Идеален для аналоговой обработки и ПИД-регулирования.
- **ST** — текстовый язык, похожий на Pascal. Самый мощный и гибкий, подходит для сложных вычислений.
- **IL** — устаревший низкоуровневый язык, больше не рекомендуется для новых проектов.
- **SFC** — язык для описания последовательных алгоритмов. Удобен для пошаговых процессов.
- На практике чаще всего комбинируют несколько языков в одном проекте.
- Для начинающих рекомендуется начать с LD, затем освоить FBD и ST.
