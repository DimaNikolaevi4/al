# Руководство по работе с Qubu.ai

Это руководство содержит пошаговые инструкции для создания, настройки и деплоя ML-модели на платформе Qubu.ai. Все чувствительные данные (логины, пароли, токены) удалены — используйте свои собственные учётные данные.

---

## 1. Авторизация на Qubu.ai

1. Перейдите на https://qubu.ai
2. Нажмите «Войти»
3. Введите email и пароль от вашего аккаунта
4. После входа вы попадёте на главную страницу с балансом коинов

---

## 2. Создание новой модели

1. Перейдите в раздел «Мои модели» → нажмите «Создать модель»
2. Заполните поля:

| Поле | Пример значения |
|------|-----------------|
| Название модели | Qwen2.5-7B-Instruct (AI Tutor) |
| Краткое описание | Система AI-тьютор для студентов СПО по специальности 15.02.14 на основе Qwen2.5-7B-Instruct с 4-bit квантизацией |
| Лицензия | Apache 2.0 |
| Публичная модель | Да |

3. Перейдите на вкладку «Метаданные»:
   - Добавьте теги: `LLM`, `СПО`, `Тьютор`, `Образование`, `qwen2.5-7b`
   - Укажите авторов
   - Отметьте участие в конкурсе «Гравитация» (если применимо)

4. Перейдите на вкладку «Техническое»:
   - Отрасль: Образование
   - Фреймворки: PyTorch, Transformers, PEFT, Safetensors
   - Языки модели: Русский, Английский

5. Нажмите «Обновить модель»

---

## 3. Загрузка кода инференса

1. Перейдите на вкладку «Инференс» модели
2. В разделе «Код» вставьте содержимое файла `service.py` из этой папки
3. В разделе «Зависимости» (requirements.txt) вставьте содержимое файла `requirements.txt`
4. Файл `bentofile.yaml` будет создан автоматически платформой

---

## 4. Настройка конфигурации инференса

### Через веб-интерфейс

1. На вкладке «Инференс» → «Конфигурация»
2. Добавьте функцию:
   - Name: `predict`
   - Path: `/predict`
   - Method: `POST`
   - Input: `message` (string, required, in body)
   - Output: showRawJson = true
   - Requires Auth: qubu

### Через API

```bash
# Установите конфигурацию инференса через PUT-запрос
curl -X PUT "https://qubu.ai/api/models/{UUID_МОДЕЛИ}/inference/config" \
  -H "Authorization: Bearer {ВАШ_BEARER_TOKEN}" \
  -H "Content-Type: application/json" \
  -d @inference_config.json
```

Где `inference_config.json` — файл из этой папки.

---

## 5. Деплой модели

1. На вкладке «Инференс» → «Деплой»
2. Выберите GPU (минимум 20 ГБ VRAM для Qwen2.5-7B 4-bit):
   - RTX A4500 (20 ГБ, ~25 коинов/час)
   - RTX 4000 Ada (20 ГБ, ~27 коинов/час)
3. Настройки автоскейла:
   - Min replicas: 1
   - Max replicas: 1
   - Терминировать при простое: Да
   - Таймаут простоя: 30 мин
4. Нажмите «Deploy»
5. Дождитесь завершения сборки (~20–25 минут для Qwen2.5-7B)

### Проверка статуса деплоя

```bash
curl -X GET "https://qubu.ai/api/models/{UUID_МОДЕЛИ}/inference/deploy" \
  -H "Authorization: Bearer {ВАШ_BEARER_TOKEN}"
```

---

## 6. Тестирование инференса

После успешного деплоя протестируйте эндпоинт:

```bash
curl -X POST "https://qubu.ai/api/models/{UUID_МОДЕЛИ}/inference/predict" \
  -H "Authorization: Bearer {ВАШ_BEARER_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"message": "Что такое ПЛК?"}'
```

Ожидаемый ответ (пример):

```json
{
  "result": {
    "answer": "ПЛК (программируемый логический контроллер) — это промышленный компьютер..."
  }
}
```

## 7. Работа с Git-репозиторием модели на Qubu

Каждая модель на Qubu имеет свой Git-репозиторий. Данные для доступа доступны на странице модели.

```bash
# Клонирование
git clone https://git.qubu.ai/{GIT_USERNAME}/{REPO_NAME}.git

# Добавление файлов
git add .
git commit -m "feat: update service code"
git push origin main
```

---

## 8. Работа с датасетом

Датасет привязывается через вкладку «Связи» на странице модели:

1. Перейдите на вкладку «Связи»
2. В выпадающем списке «Выберите датасеты» найдите ваш датасет
3. Нажмите «Привязать»
4. Сохраните изменения

## 10. Известные ограничения платформы

| Ограничение | Описание |
|-------------|----------|
| 50-минутный таймаут сборки | Модели >14 ГБ не успевают скачаться и скомпилироваться |
| Нет кэша моделей | При каждом деплое модель заново скачивается с Hugging Face |
| Нет кастомных фреймворков | Нельзя добавить bitsandbytes/accelerate в список фреймворков |
| GGUF не работает | llama-cpp-python требует компиляции, которая не укладывается в таймаут |
| Gradio нестабилен | Конфигурация Gradio часто не создаётся |
