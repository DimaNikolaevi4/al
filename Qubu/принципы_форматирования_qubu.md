# Принципы создания красивого описания для Qubu.ai

На основе практического опыта выведены рабочие правила форматирования описаний моделей и датасетов на платформе Qubu.ai.

---

## Базовые правила синтаксиса

| Что делать | Почему |
|------------|--------|
| **Теги в нижнем регистре**: `[center]`, `[b]`, `[size=5]` | Платформа не парсит заглавные теги |
| **Списки в одну строку**: `[list][*]пункт1[*]пункт2[/list]` | Избегает лишних `<p>...</p>` и пустых строк |
| **Таблицы через HTML**, а не `[table]` | BB-код таблиц не рендерится, HTML с классами — работает |
| **Весь код в одну строку** | Платформа оборачивает каждый перенос в `<p>`, что ломает вёрстку |
| **Не использовать `[hr]`** | Создаёт лишние разделители, которые сложно стилизовать |

---

## Визуальные улучшения

### Таблицы

```html
<table style="width:100%;border-collapse:collapse;border:2px solid #cbd5e1;border-radius:8px;overflow:hidden;box-shadow:0 2px 8px rgba(0,0,0,0.04)">
```

- `border:2px solid #cbd5e1` — усиленная внешняя граница
- `border-radius:8px` + `overflow:hidden` — скруглённые углы
- `box-shadow` — лёгкая тень для глубины

### Ссылки в содержании

```html
<a href="#section1" style="color:#2563eb;text-decoration:none;font-weight:500">📌 Название раздела</a>
```

- `color:#2563eb` — фирменный синий цвет
- `text-decoration:none` — убирает подчёркивание
- `font-weight:500` — делает ссылку заметнее

### Технические термины (code)

```html
<code style="background:#f1f5f9;padding:2px 6px;border-radius:4px;font-family:monospace;font-size:0.9em">Qwen2.5-7B-Instruct</code>
```

- Моноширинный шрифт + фон + скругления = профессиональный вид

## Якорные ссылки (навигация по странице)

```html
<!-- В содержании -->
<li><a href="#section1" style="color:#2563eb;text-decoration:none">📌 О проекте</a></li>

<!-- В заголовке раздела -->
<h3 id="section1">1. О проекте и проблематике</h3>
```

> **Если платформа вырезает `id="..."` из заголовков**, замените на `<div id="section1"></div>` перед заголовком.

---

## Совместимость с темами

Используйте CSS-переменные платформы для автоматической адаптации под светлую/тёмную тему:

```css
var(--bg-2)      /* фон ячеек */
var(--border)    /* границы */
var(--text-2)    /* вторичный текст */
var(--primary)   /* акцентный цвет */
```

Если переменные не поддерживаются — используйте фиксированные цвета:

```css
#cbd5e1  /* границы */
#f8fafc  /* фон заголовков таблиц */
#64748b  /* вторичный текст */
#2563eb  /* синие ссылки */
```

## Чеклист перед публикацией

- [ ] Все теги в нижнем регистре
- [ ] Списки `[list]...[/list]` без переносов внутри
- [ ] Таблицы через HTML с `border:2px` и `border-radius`
- [ ] Ссылки в содержании с `style="color:#2563eb"`
- [ ] Эмодзи добавлены логично, не перегружают текст
- [ ] Технические термины в `<code>` с фоном
- [ ] Весь код сжат в одну строку (нет лишних `\n`)
- [ ] Якоря `id="..."` и `href="#..."` совпадают
- [ ] Нет `[hr]`, `[size]`, `[font]` — только рабочий синтаксис

## Шаблон начала описания (готовый к копированию)

```html
<h2 style="text-align:center;font-weight:700;margin-bottom:0.5rem">🎓 Название модели</h2><p style="text-align:center;color:#64748b;margin-bottom:1.5rem"><strong>🤖 Краткое описание</strong></p><h3 style="text-align:center;margin-bottom:0.5rem">📋 Содержание</h3><ul style="padding-left:1.5rem;margin-bottom:1.5rem"><li><a href="#section1" style="color:#2563eb;text-decoration:none;font-weight:500">📌 Раздел 1</a></li><li><a href="#section2" style="color:#2563eb;text-decoration:none;font-weight:500">⚙️ Раздел 2</a></li></ul><h3 id="section1" style="margin-top:2rem;margin-bottom:0.75rem">1. Раздел 1</h3><p style="margin-bottom:1rem;line-height:1.6">Текст раздела...</p>
```

---

## Главный принцип

Платформа на Next.js парсит контент через `react-markdown` + собственный преобразователь. Всё, что не распознано как валидный HTML/Markdown, оборачивается в `<p>`. Поэтому **чем проще и «плосче» структура — тем стабильнее рендер**.
